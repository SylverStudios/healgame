# v0.3 — Presence, Lattice, Grace

Status: planning · Authority: active phase handoff (wins conflicts in this phase's scope) · Last verified: 2026-07-18

Ship a release that makes combat feel less punitive and more readable, makes
the talent tree a distinctive lattice (not a linear hourglass), and adds light
life to the party — music, banter, and juice — without reopening networking,
final art, procs, or hub buffs.

Reference track (background music source):
[Supervirus background score](https://soundcloud.com/sam-roberts-852591254/supervirus-background-score)
— friend-shared; **must be committed as a local asset** before the audio chunk
starts (do not stream SoundCloud at runtime).

---

## Mission

Players feel grace on near-deaths, learn boss tells visually, walk a lattice
talent tree whose lit path becomes a memorable build glyph, hear soft music
they can mute, and get short party banter on wipe / close call / victory —
while a smarter balance bot keeps the difficulty shape honest.

### Done means (user-observable)

1. Gentle looped background music plays from Hub/Tree/Combat (and settings);
   a Settings volume slider goes 0–100%; **0% fully disables** playback (no
   silent-but-decoding stream).
2. Heal casts show a clear **caster-side** animation on the healer (target juice
   already shipped — keep/complement it).
3. Wipe is a short animated transition into a **run summary** that shows last-run
   XP and a distinctive **build glyph** from the lit tree path (same summary
   pattern on victory).
4. Tank and DPS have simple attack animations (readable, not polished final art).
5. Boss abilities telegraph with a **wind-up / glow** (Dark Souls–style tells);
   the old named cast bar is no longer the primary teach — log may still name
   the ability.
6. Talent tree reads clearer: spacing, edge states, and branch-lock affordances
   are obvious at a glance.
7. Mana regen ticks have a small readable animation on the healer.
8. Tree topology is a **lattice of chains** (not the Alpha 0.2 hourglass): one
   root, multiple overlapping strings, edge locks that can destroy alternate
   routes, multiple end “crown” nodes with level gates, traversed edges light
   brightly, and the lit subgraph renders as a compact build glyph on run
   history / summary.
9. Speech bubbles: healer on close call and victory; tank on wipe; Vigil vs
   Zealot healer lines differ.
10. Coyote-time heals: after lethal damage, a **250ms** window still allows a
    heal to save the unit; death visuals wait; mid-cast heals are **not**
    cancelled solely because the target would have died at the start of that
    window.
11. Balance harness has a smarter bot that makes credible playtime decisions;
    `balance.test.ts` difficulty shape still holds (retune data if needed).

## Locked micro-choices

Central agent may refine wording but must not reopen these without updating
this handoff.

### Audio / settings

- Music file lives at `game/public/assets/audio/supervirus-background.*`
  (format: prefer `.ogg` + `.mp3` fallback, or whatever Phaser 3 accepts cleanly
  in this Vite setup — one format is fine for PoC).
- Loop; start after first user gesture if browsers require it (Settings open or
  Hub click is enough).
- Volume is a single master music slider `0..100` integer in save
  (`musicVolumePct`). **0 ⇒ stop/destroy sound, do not keep a muted channel.**
- New scene `Settings` (`SceneKeys.Settings`); Hub button to open it.
- Save bump: `healgame-save-v7` → `healgame-save-v8` (purge stale; no migration).
  Chunk H owns the bump if it ships first; any earlier chunk that needs prefs
  must coordinate so there is **one** bump for the phase.
- Journey names (add to `docs/semantic-targets.md`): `hubSettings`,
  `settingsVolumeSlider`, `settingsBack`.

### Coyote time

- Window: **250ms** constant in `game/src/data/` (not magic in engine).
- On lethal hit: unit enters `dying` (or equivalent) — not yet `dead` for
  targeting/heals; death VFX deferred.
- A heal that lands (cast completes) on a dying unit within the window restores
  them to living; death cancelled.
- If window expires with HP still ≤0: apply death as today (`unitDied`, wipe
  check).
- **Cast rules:** do **not** auto-cancel an in-flight heal solely because the
  target entered the coyote window. Cancel still happens if the target is
  *truly* dead after the window (existing `target-dead` path). Escape cancel
  unchanged.
- Mana reserve/refund rules unchanged except that successful coyote saves
  complete the cast normally (no refund).
- Amend `game/src/combat/README.md` Mid-cast target death section; add colocated
  tests. Balance bots must understand dying-but-healable targets.

### Close call (banter trigger)

- Once per combat: any living ally’s HP% drops to **≤25%** (integer HP math:
  `hp * 100 <= maxHp * 25`). Healer speech bubble fires (oath-flavored).
- Do not spam; one close-call line max per fight.

### Boss telegraphs

- Primary teach = **visual wind-up** on the boss sprite / arena cue, keyed by
  ability id (data-driven cue type: e.g. `glow`, `raise`, `pulse` — keep the
  set tiny).
- Named boss cast bar is removed or demoted (not the thing players should
  read). Combat log may still print ability names.
- Tunnel Vision focus brand may remain (already a tell); integrate, don’t
  double-signal poorly.

### Wipe / victory summary

- Replace instant “YOU WIPED / VICTORY + Return” with: short transition
  (~0.5–1.0s, temp art OK) → summary panel showing outcome, **XP gained this
  run**, and **build glyph**.
- Return still goes to Hub (`combatReturn` name preserved or aliased).
- Full multi-run history browser is **nice**; minimum is **last run** on the
  summary. Persisting a short ring buffer of runs for Hub display is allowed
  if cheap; do not build a full stats site.

### Lattice tree

- Layout metaphor (v0.3, orthogonal OK): **one root** (corner) → several
  mostly-horizontal chains left→right, plus vertical/cross links that form a
  lattice. Multiple **crown** terminals with **level requirements**.
- Edge locks remain: taking one path can destroy / exclusive-lock another edge
  (reuse `exclusiveGroup` / `requires` / existing lock machinery; extend only
  if the lattice truly needs a new edge-destroy primitive — prefer existing
  tools first).
- Traversed (owned) edges render **bright**; available dim; locked/destroyed
  clearly distinct.
- Pure function (pinned contract below) maps owned nodes/edges → a compact
  glyph descriptor for UI (letter-like silhouette is the north star — e.g. a
  “C” from vertical + two horizontals with one locked out).
- May make the tree *somewhat* easier / less linear — re-pin balance after.
- Non-orthogonal / organic layouts are **out** for v0.3 (ideas backlog).

### Banter

- Triggers: close call (healer), wipe (tank), victory (healer).
- Healer lines branch on subclass: Vigil ≈ solemn, Zealot ≈ zealous (tone,
  not comedy meme spam — dark fantasy per design-direction).
- Presentation: short speech bubble near the speaker’s unit (or portrait stand-in
  on summary if combat scene already torn down — prefer during result
  transition while sprites still exist).
- Small line tables in `game/src/data/`; no LLM, no networking.

### Juice budget

- Temp art only (rects, tweens, Kenney tiles, monospace). Reject polish creep.
- Heal caster anim: wind-up or cast pose on healer tied to `castStarted` /
  `castFinished` (complement existing target ripple/particles).
- Tank/DPS: slightly stronger or role-tinted lunge / swing — keep simple.
- Mana regen: brief blue mote or bar pulse on regen tick events (engine already
  emits or can emit a small event — prefer listening to existing mana changes
  over new systems).

### Balance bot

- Upgrade beyond current `disciplined` priority queue: better cooldown timing,
  overheal avoidance, focus-target triage, and vowstrike weaving under pressure.
- Keep styles `none` / `naive` / improved default for gates.
- When tree node ids change, update hardcoded kits in `balanceBot.ts` and
  re-verify `balance.test.ts` shape (no-heal wipe, naive wipe, crowns clear
  mid ladder, Maw unwinnable, mid-tree depth gate — amend only with evidence).

## Pinned public API contracts

Anything crossing a chunk boundary must honor these signatures (adjust only in
this handoff + central-agent commit).

### Coyote (chunk A → F/E)

```ts
// data — constant
export const COYOTE_MS = 250;

// engine view: unit is healable if alive OR within coyote window
// Events (names illustrative — match README when implemented):
//   unitDying { unitId, coyoteMs }
//   unitDied  { unitId }          // only after coyote expires unsaved
//   unitSaved { unitId }          // coyote heal landed
```

Cast cancel reason `target-dead` fires only when the target is past coyote
(truly dead), not on entering dying.

### Build glyph (chunk C → D/E)

```ts
/** Compact descriptor of the lit path — serializable, UI-agnostic. */
export type BuildGlyph = {
  /** Stable id for telemetry / equality (hash of owned edge set). */
  id: string;
  /** Grid points / segments for silhouette draw (orthogonal lattice). */
  segments: ReadonlyArray<{ x1: number; y1: number; x2: number; y2: number }>;
};

export function buildGlyphFromTree(
  config: TreeConfig,
  ownedNodeIds: ReadonlySet<string>,
): BuildGlyph;
```

UI draws `segments` only — no Phaser inside `tree/`.

### Music prefs (chunk H)

```ts
// SaveData (v8) addition:
musicVolumePct: number; // 0..100 integer; default 50 (or 0 until first gesture — pick one in chunk 0 and stick)

// Audio helper (scene-agnostic enough to call from Boot/Hub):
setMusicVolumePct(pct: number): void; // 0 stops playback
```

### Banter (chunk G)

```ts
export type BanterTrigger = 'close-call' | 'wipe' | 'victory';
export type BanterSpeaker = 'healer' | 'tank';

export function pickBanterLine(args: {
  trigger: BanterTrigger;
  speaker: BanterSpeaker;
  subclass: 'vigil' | 'zealot' | null;
  rng?: () => number; // inject for tests; default deterministic pick by index
}): string;
```

Engine stays pure — close-call detection may live in scene **or** a pure helper
fed by HP snapshots; prefer a pure helper with tests.

- Full run-history analytics UI / cloud sync (telemetry CLI already exists).
- Replacing Kenney combat tiles.
- Rewriting the entire combat engine or abandoning `advance(dtMs)` determinism.
